## 🎤 Work Story

- ### 프로젝트 투입 배경

  4월에 오픈을 한 한화생명 app 에서 *7월에 오픈 할 구독 보험 상품 관련 화면 퍼블리싱*을 위해 투입하여 프로젝트를 진행 하였습니다.

- ### 프로젝트 이슈 및 대응 사항

  1. 사이트가 오픈이 된 시점 인데 개발팀에 퍼블리싱 파일을 _어떻게 작업_ 해서 _전달_ 할지 _명확하게 규칙_ 이 정해져 있지 _않은_ 상태 였습니다.
     > 마크업 작업을 _기존에 작업 했던대로_ 하라는 의견과 _기존 작업 대로 하지 말고_ html,css 파일로 작업 해서 개발팀에 전달 해 달라는 의견 이렇게 2개의 주장이 정리가 되지 않은 상태 였는데 저의 입장에서는 정해 지기만 하면 어느 쪽이든 상관이 없는 상황이라 *실무자(개발팀, 퍼블팀, 디자인팀) 들께 정리를 다시 한번 해달라고 요청*을 드렸고, 기존 작업 했던 대로 *styled component*를 사용해서 마크업 하되 _타입스크립트 작업은 제외_ 하고 개발팀에 전달 하기로 결정을 지었습니다.
  2. *전체 페이지*에서 *사용*하는 버튼, 제목, 같은 *공통 UI 요소*들을 _공통 클래스명_ 작업 _없이_ 진행이 된 상태 였고, 개발팀에서 추가로 *요청*한 사항 중에 _특정 화면_ 컴포넌트를 *다른 페이지*에 _붙였을_ 때 _깨지지 않도록_ 해달라는 요청사항을 받았습니다.
     > styled component를 *전역*에서 _사용 할 수_ 있도록 *조치*를 취한 다음 _공통 ui 요소_ 들을 *전체 페이지*에서 _사용 할 수_ 있도록 퍼블리싱을 하였고 CSS 클래스명 작명법 중 _BEM_ 방법을 활용 하여 UI 요소 컴포넌트 별로 클래스명 작업을 했기 때문에 _ui 요소 들끼리 클래스명이 꼬이지_ 않도록 작업 했고, 그렇기 때문에 _다른 페이지에서 붙였을 때_ 깨지지 않게끔 작업 하였습니다.
